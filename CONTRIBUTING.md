# Contributing to AI Behavioral Safety Studies

Thank you for your interest in contributing to this portfolio of AI safety evaluations. This document outlines guidelines for contributing new case studies, methodology improvements, or observations.

## Contribution Areas

### 1. New Case Studies
We welcome new case studies that:
- Investigate different framing strategies
- Test across diverse model types
- Examine various content domains
- Explore cultural or linguistic variations

### 2. Methodology Improvements
Contributions to the evaluation framework:
- Enhanced testing protocols
- New metrics or analysis techniques
- Improved ethical guidelines
- Better reproducibility practices

### 3. Observations & Synthesis
- Cross-study pattern identification
- Meta-analysis of findings
- Theoretical interpretations
- Practical recommendations

### 4. Documentation & Tooling
- Improved documentation
- Analysis tools or scripts
- Visualization approaches
- Educational materials

## Contribution Process

### 1. Pre-Contribution Checklist
- [ ] Review existing case studies to avoid duplication
- [ ] Ensure ethical guidelines are followed
- [ ] Plan for reproducible methodology
- [ ] Consider anonymization requirements

### 2. Submission Format
New case studies should follow the established structure:
```
case-studies/case-XX-topic-description.md
```
With sections:
- Executive Summary
- Study Design
- Detailed Methodology
- Results
- Analysis
- Discussion
- Recommendations
- References

### 3. Quality Standards
- **Clarity**: Well-organized, clearly written
- **Reproducibility**: Detailed methodology, transparent process
- **Ethical compliance**: Follows established guidelines
- **Evidence-based**: Data-supported conclusions
- **Anonymization**: No personally identifiable information

### 4. Review Process
1. **Initial review**: Methodology and ethical compliance
2. **Technical review**: Reproducibility and analysis validity
3. **Final review**: Documentation quality and contribution value
4. **Integration**: Merged into appropriate portfolio section

## Ethical Guidelines

### Required Practices
1. **No harm**: Avoid generating harmful, unethical, or illegal content
2. **Anonymization**: Remove all personally identifiable information
3. **Transparency**: Clear documentation of methods and limitations
4. **Responsible disclosure**: Serious issues reported to appropriate parties
5. **Academic integrity**: Proper attribution and citation

### Prohibited Content
- Personally identifiable information
- Disallowed or harmful content generation
- Unethical testing practices
- Misleading or false claims
- Unsubstantiated conclusions

## Technical Standards

### Documentation
- Markdown formatting
- Clear section structure
- Consistent terminology
- Complete references

### Data Management
- Anonymized transcripts only
- No sensitive or private data
- Appropriate file organization
- Clear data provenance

### Analysis
- Transparent methodology
- Appropriate statistical methods
- Clear separation of observation and interpretation
- Consideration of limitations

## Getting Started

### For New Contributors
1. Read the existing case studies and methodology
2. Review the ethical guidelines thoroughly
3. Start with a small, focused case study
4. Seek feedback early in the process

### Suggested First Contributions
1. Replication of existing studies with different models
2. Extension of framing strategies to new domains
3. Development of analysis tools or metrics
4. Improvement of documentation or examples

## Communication

### Questions & Discussion
- Open issues for methodology questions
- Discuss ethical considerations before testing
- Seek clarification on documentation standards
- Share preliminary findings for feedback

### Feedback Process
- Constructive, evidence-based feedback
- Focus on methodology and ethics
- Respect diverse perspectives
- Collaborative improvement orientation
